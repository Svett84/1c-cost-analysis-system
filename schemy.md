Название процесса
Расчет себестоимости продукции, учет брака и формирование аналитического отчета.

1. Инициализация
На первом шаге происходит очистка (инициализация) регистра «ПК_Выработка».

Действие: Удаляются старые (ранее сохраненные) записи.

Цель: Подготовка хранилища данных для записи новых итоговых показателей по текущему документу (например, за смену или партию).

2. Основной цикл перебора продукции
Система последовательно обрабатывает каждую строку из таблицы «Продукция» (например, различные номенклатурные позиции). Для каждой позиции выполняются следующие шаги:

2.1. Расчет трудозатрат (часов)
Вычисляется Общее количество выпущенных единиц (годные + брак).

Рассчитывается ЧасыВсего = Общее количество × Норма времени на единицу.

2.2. Запрос стоимости нормо-часа и расчет плановой суммы
Выполняется запрос к регистру сведений «ПК_СтоимостьНормоЧаса».

Условие: Данные берутся строго на дату текущего документа.

Расчет: СуммаПлан = ЧасыВсего × Цена нормо-часа (актуальная на дату).

2.3. Расчет стоимости комплектующих
Рассчитывается стоимость использованных запчастей для годной продукции:

СуммаЗапчастей = Цена запчастей × Количество годного.

2.4. Вложенный цикл по браку (детальный учет)
Для учета порчи комплектующиых запускается внутренний цикл по таблице «ДеталиБрака»:

Условие проверки: Проверяется совпадение велосипеда (или иного изделия) — вероятно, идентификация, к какому именно изделию из основного цикла относится бракованная деталь.

Накопление суммы брака: Если условие выполняется, сумма брака увеличивается:

СуммаБрака = СуммаБрака + (Количество брака × Цена детали).

3. Сохранение результата
После завершения расчетов по конкретной строке продукции (включая обработку всех связанных деталей брака) данные записываются в регистр накопления или сведений «ПК_Выработка».

Состав записи: В регистр, вероятно, попадают итоговые показатели: Общее время, Плановая сумма, Стоимость запчастей, Сумма брака и себестоимость.

4. Завершение и формирование отчета
После того как все строки из таблицы «Продукция» обработаны и данные сохранены, система переходит к заключительному этапу:

На основе накопленных данных в регистре «ПК_Выработка» запускается механизм СКФ (Система Компоновки Данных) .

Результат: Формируется итоговый отчет «Анализ выпуска и брака» , который выводится пользователю для анализа эффективности производства и потерь от брака.
